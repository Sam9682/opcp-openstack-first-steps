Aide-mémoire CLI & API

Difficulté : Débutant · Durée estimée : 10 min

Cette page liste les opérations OpenStack courantes avec le CLI openstack et les appels curl équivalents vers l'API REST. Remplacez les valeurs de substitution ($OS_AUTH_URL, $TOKEN, etc.) par vos propres variables d'environnement.

Authentification — Obtenir un token

CLI

openstack token issue

curl — Application Credentials

curl -s -X POST "$OS_AUTH_URL/auth/tokens" \
  -H "Content-Type: application/json" \
  -d '{
    "auth": {
      "identity": {
        "methods": ["application_credential"],
        "application_credential": {
          "id": "'"$OS_APPLICATION_CREDENTIAL_ID"'",
          "secret": "'"$OS_APPLICATION_CREDENTIAL_SECRET"'"
        }
      }
    }
  }' -i | grep "X-Subject-Token"

curl — Utilisateur / Mot de passe

curl -s -X POST "$OS_AUTH_URL/auth/tokens" \
  -H "Content-Type: application/json" \
  -d '{
    "auth": {
      "identity": {
        "methods": ["password"],
        "password": {
          "user": {
            "name": "'"$OS_USERNAME"'",
            "domain": {"name": "'"$OS_DOMAIN_NAME"'"},
            "password": "'"$OS_PASSWORD"'"
          }
        }
      },
      "scope": {
        "project": {
          "name": "'"$OS_PROJECT_NAME"'",
          "domain": {"name": "'"$OS_DOMAIN_NAME"'"}
        }
      }
    }
  }' -i | grep "X-Subject-Token"

Compute (Nova)

Lister les serveurs — CLI

openstack server list

Lister les serveurs — curl

curl -s -X GET "$NOVA_ENDPOINT/servers" \
  -H "X-Auth-Token: $TOKEN" | python3 -m json.tool

Créer un serveur — CLI

openstack server create \
  --flavor s1-4 \
  --image "Ubuntu 22.04" \
  --network my-network \
  --wait \
  my-instance

Créer un serveur — curl

curl -s -X POST "$NOVA_ENDPOINT/servers" \
  -H "X-Auth-Token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "server": {
      "name": "my-instance",
      "flavorRef": "'"$FLAVOR_ID"'",
      "imageRef": "'"$IMAGE_ID"'",
      "networks": [{"uuid": "'"$NETWORK_ID"'"}]
    }
  }' | python3 -m json.tool

Supprimer un serveur — CLI

openstack server delete my-instance

Supprimer un serveur — curl

curl -s -X DELETE "$NOVA_ENDPOINT/servers/$SERVER_ID" \
  -H "X-Auth-Token: $TOKEN"

Réseau (Neutron)

Lister les réseaux — CLI

openstack network list

Lister les réseaux — curl

curl -s -X GET "$NEUTRON_ENDPOINT/v2.0/networks" \
  -H "X-Auth-Token: $TOKEN" | python3 -m json.tool

Créer un réseau — CLI

openstack network create my-network

Créer un réseau — curl

curl -s -X POST "$NEUTRON_ENDPOINT/v2.0/networks" \
  -H "X-Auth-Token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "network": {
      "name": "my-network",
      "admin_state_up": true
    }
  }' | python3 -m json.tool

Créer un sous-réseau — CLI

openstack subnet create \
  --network my-network \
  --subnet-range 192.168.1.0/24 \
  my-subnet

Créer un sous-réseau — curl

curl -s -X POST "$NEUTRON_ENDPOINT/v2.0/subnets" \
  -H "X-Auth-Token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "subnet": {
      "network_id": "'"$NETWORK_ID"'",
      "ip_version": 4,
      "cidr": "192.168.1.0/24",
      "name": "my-subnet"
    }
  }' | python3 -m json.tool

Créer un volume — curl

Attacher un volume à un serveur — CLI

Attacher un volume à un serveur — curl

curl -s -X POST "$NOVA_ENDPOINT/servers/$SERVER_ID/os-volume_attachments" \
  -H "X-Auth-Token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "volumeAttachment": {
      "volumeId": "'"$VOLUME_ID"'"
    }
  }' | python3 -m json.tool

Astuces utiles

  • Stocker votre token dans une variable pour le réutiliser : 
    export TOKEN=$(openstack token issue -f value -c id)
  • Trouver les endpoints des services : 
    openstack catalog list
  • Utiliser -f json avec le CLI pour obtenir une sortie JSON, pratique pour le scripting : 
    openstack server list -f json
