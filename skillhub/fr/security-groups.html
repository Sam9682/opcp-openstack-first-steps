Groupes de sécurité

Difficulté : Intermédiaire · Durée estimée : 20 min

Les groupes de sécurité agissent comme des pare-feu virtuels pour vos instances. Dans cette leçon, vous allez créer des groupes de sécurité, gérer les règles d'entrée/sortie et les appliquer aux instances.

1. Exercice 1 — Créer un groupe de sécurité

Utilisez security_group_manager pour créer un groupe de sécurité nommé {student_id}-sg-web. 

sg = self.managers.get("security_group_manager")

group = sg.create(
    name=f"{student_id}-sg-web",
    description="Groupe de sécurité pour le trafic web",
)
print(f"Groupe de sécurité créé : {group}")

2. Exercice 2 — Gérer les règles

Ajoutez des règles d'entrée pour autoriser le trafic HTTP (port 80) et HTTPS (port 443). Les règles sont validées avant d'être envoyées à l'API. 

from labs.core.validators import validate_sg_rule

valid, err = validate_sg_rule("tcp", 443, 443)
if not valid:
    print(f"Validation échouée : {err}")

sg.add_rule(
    f"{student_id}-sg-web",
    protocol="tcp",
    port_min=443,
    port_max=443,
    direction="ingress",
)
print("Règle HTTPS ajoutée")

3. Exercice 3 — Appliquer à une instance

Appliquez votre groupe de sécurité à une instance en cours d'exécution. Cela contrôle quel trafic réseau peut atteindre l'instance. 

sg.apply_to_instance(
    sg_name=f"{student_id}-sg-web",
    instance_name=f"{student_id}-compute-instance",
)
print("Groupe de sécurité appliqué à l'instance")

Résumé

  • Les groupes de sécurité sont des pare-feu virtuels qui contrôlent l'accès réseau.
  • Les règles spécifient le protocole, la plage de ports et la direction (entrée/sortie).
  • Les règles sont validées avant d'être envoyées à l'API.
  • Les groupes de sécurité sont appliqués aux instances pour appliquer les politiques réseau.
