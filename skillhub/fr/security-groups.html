Groupes de sécurité

Difficulté : Intermédiaire · Durée estimée : 20 min

Les groupes de sécurité agissent comme des pare-feu virtuels pour vos instances. Dans cette leçon, vous allez créer des groupes de sécurité, gérer les règles d'entrée/sortie et les appliquer aux instances.

1. Exercice 1 — Créer un groupe de sécurité

Utilisez le CLI openstack pour créer un groupe de sécurité nommé {student_id}-sg-web. 

# Créer un groupe de sécurité
openstack security group create \
  --description "Groupe de sécurité pour le trafic web" \
  {student_id}-sg-web

# Vérifier que le groupe de sécurité a été créé
openstack security group show {student_id}-sg-web

2. Exercice 2 — Gérer les règles

Ajoutez des règles d'entrée pour autoriser le trafic HTTP (port 80) et HTTPS (port 443). 

# Ajouter une règle d'entrée HTTP (port 80)
openstack security group rule create \
  --protocol tcp \
  --dst-port 80 \
  --ingress \
  {student_id}-sg-web

# Ajouter une règle d'entrée HTTPS (port 443)
openstack security group rule create \
  --protocol tcp \
  --dst-port 443 \
  --ingress \
  {student_id}-sg-web

# Lister les règles pour vérifier
openstack security group rule list {student_id}-sg-web

3. Exercice 3 — Appliquer à une instance

Appliquez votre groupe de sécurité à une instance en cours d'exécution. Cela contrôle quel trafic réseau peut atteindre l'instance. 

# Ajouter le groupe de sécurité à une instance
openstack server add security group \
  {student_id}-compute-instance \
  {student_id}-sg-web

# Vérifier que le groupe de sécurité est attaché
openstack server show {student_id}-compute-instance -f value -c security_groups

Résumé

  • Les groupes de sécurité sont des pare-feu virtuels qui contrôlent l'accès réseau.
  • Les règles spécifient le protocole, la plage de ports et la direction (entrée/sortie).
  • Utilisez openstack security group rule create pour ajouter des règles.
  • Les groupes de sécurité sont appliqués aux instances avec openstack server add security group.
