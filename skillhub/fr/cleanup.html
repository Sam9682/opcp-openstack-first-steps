Nettoyage des ressources

Difficulté : Débutant · Durée estimée : 10 min

Maintenant que vous avez terminé tous les labs, il est temps de nettoyer toutes les ressources OpenStack créées pendant les exercices. Toutes les ressources suivent la convention de nommage ${STUDENT_ID}-*, ce qui les rend faciles à identifier et supprimer.

1. Convention de nommage

Chaque ressource créée au cours des labs est préfixée par votre student_id. Voici la liste complète :

ModuleType de ressourceNom
Premiers pasInstance${STUDENT_ID}-first-instance
Premiers pasRéseau${STUDENT_ID}-first-network
Premiers pasVolume${STUDENT_ID}-first-volume

LACPPorts${STUDENT_ID}-lacp-port-1, ${STUDENT_ID}-lacp-port-2
LACPBond${STUDENT_ID}-lacp-bond
LACPInstance${STUDENT_ID}-lacp-instance
SetupRéseaux${STUDENT_ID}-compute-network, ${STUDENT_ID}-storage-network, ${STUDENT_ID}-sg-network

2. Ordre de suppression

Les ressources OpenStack ont des dépendances. Les supprimer dans le mauvais ordre provoquera des erreurs. L'ordre correct est :

  1. Détacher les volumes des instances (Cinder).
  2. Supprimer les snapshots — snapshots d'instances (Glance/Nova) et de volumes (Cinder).
  3. Supprimer les instances (Nova).
  4. Supprimer les bonds et ports (Neutron).
  5. Supprimer les volumes (Cinder).
  6. Supprimer les groupes de sécurité (Neutron).
  7. Supprimer les réseaux (Neutron) — les sous-réseaux sont supprimés automatiquement.

3. Nettoyage automatisé

Le framework de lab fournit un script de nettoyage qui supprime toutes les ressources correspondant au préfixe ${STUDENT_ID}-* dans le bon ordre de dépendances : 

# Nettoyer toutes les ressources de votre session
python3 -m labs.scripts.cleanup_lab --student-id votre-student-id

Le script utilise une approche best-effort : si une ressource ne peut pas être supprimée, l'erreur est journalisée et le script continue avec les ressources restantes.

4. Nettoyage manuel avec le CLI

Vous pouvez aussi nettoyer les ressources manuellement avec le CLI openstack. Suivez l'ordre de suppression ci-dessus : 

# 1. Détacher les volumes des instances
openstack server remove volume ${STUDENT_ID}-compute-instance ${STUDENT_ID}-storage-volume

# 2. Supprimer les snapshots (instance et volume)
openstack image delete ${STUDENT_ID}-compute-snapshot
openstack volume snapshot delete ${STUDENT_ID}-storage-snapshot

# 3. Supprimer les instances
openstack server delete ${STUDENT_ID}-first-instance
openstack server delete ${STUDENT_ID}-compute-instance
openstack server delete ${STUDENT_ID}-lacp-instance

# 4. Supprimer les trunks et ports
openstack network trunk delete ${STUDENT_ID}-lacp-bond
openstack port delete ${STUDENT_ID}-lacp-port-1
openstack port delete ${STUDENT_ID}-lacp-port-2

# 5. Supprimer les volumes
openstack volume delete ${STUDENT_ID}-first-volume
openstack volume delete ${STUDENT_ID}-storage-volume

# 6. Supprimer les groupes de sécurité
openstack security group delete ${STUDENT_ID}-sg-web
openstack security group delete ${STUDENT_ID}-default-sg

# 7. Supprimer les routeurs (retirer les interfaces d'abord) et les réseaux
openstack router remove subnet ${STUDENT_ID}-net-router ${STUDENT_ID}-net-subnet
openstack router delete ${STUDENT_ID}-net-router
openstack network delete ${STUDENT_ID}-first-network
openstack network delete ${STUDENT_ID}-net-network
openstack network delete ${STUDENT_ID}-lacp-network
openstack network delete ${STUDENT_ID}-compute-network
openstack network delete ${STUDENT_ID}-storage-network
openstack network delete ${STUDENT_ID}-sg-network

5. Vérifier le nettoyage

Après le nettoyage, vérifiez qu'il ne reste aucune ressource en les listant avec le CLI OpenStack : 

# Lister les instances restantes
openstack server list --name "votre-student-id-*"

# Lister les volumes restants
openstack volume list --name "votre-student-id-*"

# Lister les réseaux restants
openstack network list --name "votre-student-id-*"

# Lister les groupes de sécurité restants
openstack security group list --project votre-project-id | grep "votre-student-id"

Si des ressources subsistent, supprimez-les manuellement avec le CLI openstack ou relancez le script de nettoyage.
