Configuration LACP

Difficulté : Avancé · Durée estimée : 30 min

Le protocole LACP (Link Aggregation Control Protocol) permet de regrouper plusieurs interfaces réseau physiques en un seul lien logique, offrant une bande passante accrue et de la redondance. Dans cette leçon, vous allez configurer des bonds LACP sur OpenStack via l'API Neutron.

Contexte — Qu'est-ce que le LACP ?

Le LACP (IEEE 802.3ad) négocie automatiquement l'agrégation de liens entre deux points. Dans un contexte OpenStack, il est généralement utilisé pour agréger des ports sur des instances bare-metal ou SR-IOV, ou pour configurer des ports trunk transportant plusieurs VLANs sur un lien agrégé.

  • Mode actif — l'interface envoie des trames LACPDU pour initier la négociation.
  • Mode passif — l'interface répond aux trames LACPDU sans les initier.
  • Modes de bond — balance-rr, active-backup, balance-xor, 802.3ad (LACP).

1. Exercice 1 — Créer un port bond LACP

Utilisez network_manager pour créer deux ports Neutron sur le même réseau, puis créez un port trunk qui les agrège en utilisant le mode LACP (802.3ad). 

net = self.managers.get("network_manager")

# Créer le réseau de base
network = net.create(name=f"{student_id}-lacp-network")

# Créer deux ports membres
port1 = net.create_port(
    network_name=f"{student_id}-lacp-network",
    name=f"{student_id}-lacp-port-1",
)
port2 = net.create_port(
    network_name=f"{student_id}-lacp-network",
    name=f"{student_id}-lacp-port-2",
)

# Créer le bond LACP
bond = net.create_bond(
    name=f"{student_id}-lacp-bond",
    port_names=[f"{student_id}-lacp-port-1", f"{student_id}-lacp-port-2"],
    bond_mode="802.3ad",
)
print(f"Bond LACP créé : {bond}")

2. Exercice 2 — Configurer les paramètres LACP

Ajustez le bond LACP en définissant le débit LACP, la politique de hachage de transmission et le mode d'activité LACP. Ces paramètres contrôlent la négociation et la distribution du trafic. 

# Mettre à jour les paramètres LACP du bond
net.update_bond(
    bond_name=f"{student_id}-lacp-bond",
    lacp_rate="fast",              # fast = 1s, slow = 30s
    transmit_hash_policy="layer3+4",  # hachage sur IP + port
    lacp_activity="active",        # négociation LACPDU active
)
print("Paramètres LACP configurés")

3. Exercice 3 — Attacher le bond LACP à une instance

Attachez le bond LACP à une instance de calcul afin que celle-ci bénéficie de la bande passante agrégée et de la redondance de lien. 

compute = self.managers.get("compute_manager")

# Lancer une instance
instance = compute.create(
    name=f"{student_id}-lacp-instance",
    image="ubuntu-22.04",
    flavor="s1-4",
)

# Attacher le bond à l'instance
net.attach_bond_to_instance(
    bond_name=f"{student_id}-lacp-bond",
    instance_name=f"{student_id}-lacp-instance",
)
print(f"Bond attaché à l'instance : {instance}")

Résumé

  • Le LACP (802.3ad) agrège plusieurs ports en un seul lien logique.
  • Utilisez network_manager pour créer des ports, des bonds et configurer les paramètres LACP.
  • Paramètres clés : lacp_rate (fast/slow), transmit_hash_policy et lacp_activity (active/passive).
  • Les bonds peuvent être attachés aux instances pour une bande passante accrue et du basculement automatique.
