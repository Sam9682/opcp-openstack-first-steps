Configuration LACP

Difficulté : Avancé · Durée estimée : 30 min

Le protocole LACP (Link Aggregation Control Protocol) permet de regrouper plusieurs interfaces réseau physiques en un seul lien logique, offrant une bande passante accrue et de la redondance. Dans cette leçon, vous allez configurer des bonds LACP sur OpenStack via l'API Neutron.

Contexte — Qu'est-ce que le LACP ?

Le LACP (IEEE 802.3ad) négocie automatiquement l'agrégation de liens entre deux points. Dans un contexte OpenStack, il est généralement utilisé pour agréger des ports sur des instances bare-metal ou SR-IOV, ou pour configurer des ports trunk transportant plusieurs VLANs sur un lien agrégé.

  • Mode actif — l'interface envoie des trames LACPDU pour initier la négociation.
  • Mode passif — l'interface répond aux trames LACPDU sans les initier.
  • Modes de bond — balance-rr, active-backup, balance-xor, 802.3ad (LACP).

1. Exercice 1 — Créer un port bond LACP

Utilisez le CLI openstack pour créer deux ports Neutron sur le même réseau, puis créez un port trunk qui les agrège en utilisant le mode LACP (802.3ad). 

# Créer le réseau de base
openstack network create {student_id}-lacp-network

# Créer deux ports membres sur le réseau
openstack port create \
  --network {student_id}-lacp-network \
  {student_id}-lacp-port-1

openstack port create \
  --network {student_id}-lacp-network \
  {student_id}-lacp-port-2

# Créer un trunk port en utilisant le premier port comme parent
openstack network trunk create \
  --parent-port {student_id}-lacp-port-1 \
  --subport port={student_id}-lacp-port-2,segmentation-type=vlan,segmentation-id=100 \
  {student_id}-lacp-bond

# Vérifier que le trunk a été créé
openstack network trunk show {student_id}-lacp-bond

2. Exercice 2 — Configurer les paramètres LACP

Ajustez le bond LACP en mettant à jour la configuration du trunk port. Ces paramètres contrôlent la négociation et la distribution du trafic. 

# Ajouter un autre subport au trunk
openstack network trunk set \
  --subport port={student_id}-lacp-port-2,segmentation-type=vlan,segmentation-id=200 \
  {student_id}-lacp-bond

# Vérifier la configuration du trunk
openstack network trunk show {student_id}-lacp-bond

3. Exercice 3 — Attacher le bond LACP à une instance

Attachez le bond LACP à une instance de calcul afin que celle-ci bénéficie de la bande passante agrégée et de la redondance de lien. 

# Lancer une instance en utilisant le port parent du trunk
openstack server create \
  --flavor s1-4 \
  --image "ubuntu-22.04" \
  --port {student_id}-lacp-port-1 \
  --wait \
  {student_id}-lacp-instance

# Vérifier que l'instance est active et connectée
openstack server show {student_id}-lacp-instance -f value -c status
openstack port show {student_id}-lacp-port-1 -f value -c status

Résumé

  • Le LACP (802.3ad) agrège plusieurs ports en un seul lien logique.
  • Utilisez les commandes openstack network trunk pour créer et gérer les trunk ports.
  • Les subports sont ajoutés avec --subport en spécifiant le type et l'ID de segmentation.
  • Les instances se connectent aux trunks via le port parent avec --port.
