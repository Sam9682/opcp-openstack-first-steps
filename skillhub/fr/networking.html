Réseau

Difficulté : Intermédiaire · Durée estimée : 20 min

Les réseaux sont la base de tout déploiement cloud. Dans cette leçon, vous allez créer des réseaux virtuels, configurer des sous-réseaux avec validation CIDR et mettre en place des routeurs via l'API Neutron.

1. Exercice 1 — Créer un réseau

Utilisez le CLI openstack pour créer un réseau virtuel nommé ${STUDENT_ID}-net-network. 

# Créer un réseau virtuel
openstack network create ${STUDENT_ID}-net-network

# Vérifier que le réseau a été créé
openstack network show ${STUDENT_ID}-net-network -f value -c status

2. Exercice 2 — Créer un sous-réseau

Créez un sous-réseau dans votre réseau. Utilisez 10.0.0.0/24 comme plage d'adresses. 

# Créer un sous-réseau avec le CIDR 10.0.0.0/24
openstack subnet create \
  --network ${STUDENT_ID}-net-network \
  --subnet-range 10.0.0.0/24 \
  ${STUDENT_ID}-net-subnet

# Vérifier que le sous-réseau a été créé
openstack subnet show ${STUDENT_ID}-net-subnet

3. Réseaux VLAN et ports

En environnement de production, les réseaux sont souvent adossés à des VLANs spécifiques. Vous pouvez créer un réseau VLAN et y assigner un port afin que le trafic soit tagué avec le bon VLAN ID sur le switch physique.

Créer un réseau VLAN

# Créer un réseau avec un VLAN ID spécifique
openstack network create \
  --provider-network-type vlan \
  --provider-physical-network physnet1 \
  --provider-segment 100 \
  ${STUDENT_ID}-vlan100-network

# Créer un sous-réseau sur le réseau VLAN
openstack subnet create \
  --network ${STUDENT_ID}-vlan100-network \
  --subnet-range 10.100.0.0/24 \
  ${STUDENT_ID}-vlan100-subnet

Créer un port sur un réseau VLAN spécifique

# Créer un port lié au réseau VLAN
openstack port create \
  --network ${STUDENT_ID}-vlan100-network \
  ${STUDENT_ID}-vlan100-port

# Vérifier les détails du port (binding VLAN)
openstack port show ${STUDENT_ID}-vlan100-port

4. Assignation VLAN pour le baremetal

Lorsque vous travaillez avec des instances baremetal (Ironic), vous pouvez assigner des réseaux VLAN directement aux interfaces physiques. Cela permet d'isoler le trafic sur du matériel dédié. 

# Créer un réseau VLAN pour le baremetal
openstack network create \
  --provider-network-type vlan \
  --provider-physical-network physnet1 \
  --provider-segment 200 \
  ${STUDENT_ID}-bm-vlan200

# Créer un sous-réseau
openstack subnet create \
  --network ${STUDENT_ID}-bm-vlan200 \
  --subnet-range 10.200.0.0/24 \
  ${STUDENT_ID}-bm-vlan200-subnet

# Créer un port pour le nœud baremetal
openstack port create \
  --network ${STUDENT_ID}-bm-vlan200 \
  --vnic-type baremetal \
  ${STUDENT_ID}-bm-port

# Déployer une instance baremetal avec ce port
openstack server create \
  --flavor baremetal \
  --image "Ubuntu 22.04" \
  --port ${STUDENT_ID}-bm-port \
  --wait \
  ${STUDENT_ID}-bm-instance

5. Ports trunk

Les ports trunk permettent à un seul port parent de transporter le trafic de plusieurs réseaux VLAN via des subports. C'est couramment utilisé lorsqu'une instance a besoin d'accéder à plusieurs VLANs simultanément. 

# Créer le port parent
openstack port create \
  --network ${STUDENT_ID}-net-network \
  ${STUDENT_ID}-trunk-parent

# Créer un subport sur un autre réseau VLAN
openstack port create \
  --network ${STUDENT_ID}-vlan100-network \
  ${STUDENT_ID}-trunk-sub1

# Créer le trunk avec le port parent et attacher le subport
openstack network trunk create \
  --parent-port ${STUDENT_ID}-trunk-parent \
  --subport port=${STUDENT_ID}-trunk-sub1,segmentation-type=vlan,segmentation-id=100 \
  ${STUDENT_ID}-trunk

# Ajouter un autre subport ultérieurement
openstack port create \
  --network ${STUDENT_ID}-bm-vlan200 \
  ${STUDENT_ID}-trunk-sub2

openstack network trunk set \
  --subport port=${STUDENT_ID}-trunk-sub2,segmentation-type=vlan,segmentation-id=200 \
  ${STUDENT_ID}-trunk

# Vérifier la configuration du trunk
openstack network trunk show ${STUDENT_ID}-trunk

6. QinQ (802.1ad) — Double encapsulation VLAN

Le QinQ (aussi appelé VLAN stacking ou 802.1ad) encapsule un tag VLAN à l'intérieur d'un autre. C'est utile lorsqu'un fournisseur de services doit transporter le trafic VLAN de ses clients à travers sa propre infrastructure VLAN sans conflit. Le tag externe (S-VLAN) identifie le segment du fournisseur, tandis que le tag interne (C-VLAN) préserve le VLAN original du client.

Cas d'usage

Un hébergeur exploite plusieurs locataires sur la même infrastructure physique. Chaque locataire utilise ses propres VLANs en interne (ex. VLAN 10, 20, 30). Avec le QinQ, le fournisseur encapsule le trafic de chaque locataire dans un VLAN externe unique, permettant aux VLAN IDs des locataires de se chevaucher sans interférence. 

# Créer un réseau fournisseur avec le type VLAN pour le tag externe (S-VLAN)
openstack network create \
  --provider-network-type vlan \
  --provider-physical-network physnet1 \
  --provider-segment 500 \
  ${STUDENT_ID}-qinq-provider

# Créer un sous-réseau sur le réseau fournisseur
openstack subnet create \
  --network ${STUDENT_ID}-qinq-provider \
  --subnet-range 10.50.0.0/24 \
  ${STUDENT_ID}-qinq-provider-subnet

# Créer le port parent sur le réseau fournisseur
openstack port create \
  --network ${STUDENT_ID}-qinq-provider \
  ${STUDENT_ID}-qinq-parent

# Créer un trunk et attacher un subport avec le tag interne (C-VLAN)
openstack network trunk create \
  --parent-port ${STUDENT_ID}-qinq-parent \
  --subport port=${STUDENT_ID}-trunk-sub1,segmentation-type=vlan,segmentation-id=10 \
  ${STUDENT_ID}-qinq-trunk

# L'instance voit le trafic du VLAN interne 10,
# tandis que le réseau physique le transporte dans le VLAN externe 500
openstack network trunk show ${STUDENT_ID}-qinq-trunk

Résumé

  • Utilisez les commandes openstack network et openstack subnet.
  • Les sous-réseaux nécessitent une plage CIDR valide (ex. 10.0.0.0/24).
  • Les réseaux VLAN utilisent --provider-network-type vlan avec un --provider-segment ID.
  • Les ports baremetal utilisent --vnic-type baremetal pour se lier aux interfaces physiques.
  • Les ports trunk transportent plusieurs VLANs via les commandes openstack network trunk.
  • Le QinQ (802.1ad) empile un C-VLAN interne dans un S-VLAN externe pour l'isolation multi-locataire.
