Avant de plonger dans les services individuels, passons en revue les concepts fondamentaux d'OpenStack et découvrons les différentes façons d'interagir avec la plateforme pour créer des réseaux, des instances, des volumes, etc.

1. Concepts fondamentaux

Projets

Un projet (aussi appelé tenant) est une unité organisationnelle dans OpenStack. Les ressources comme les instances, réseaux et volumes appartiennent à un projet.

Services

OpenStack est composé de services indépendants :

  • Nova — Calcul (instances de machines virtuelles)
  • Neutron — Réseau (réseaux virtuels, sous-réseaux, routeurs)
  • Cinder — Stockage bloc (volumes, snapshots)
  • Keystone — Identité (authentification, autorisation)
  • Glance — Image (images de machines virtuelles)

Endpoints

Chaque service expose des endpoints HTTP listés dans le catalogue de services Keystone. Les clients découvrent les URLs des services en interrogeant le catalogue après authentification.

2. Accéder à l'environnement OpenStack

OpenStack offre plusieurs moyens de gérer votre infrastructure. Selon votre cas d'usage — opérations manuelles rapides, scripting ou infrastructure as code — vous pouvez choisir l'approche la plus adaptée.

Option A — API REST (cURL / HTTP)

Chaque service OpenStack expose une API REST standard. Vous pouvez interagir directement avec elle via curl ou tout client HTTP. C'est l'approche la plus bas niveau, qui donne un contrôle total sur chaque requête. 

# 1. S'authentifier et obtenir un token
TOKEN=$(curl -si -X POST "$OS_AUTH_URL/auth/tokens" \
  -H "Content-Type: application/json" \
  -d '{
    "auth": {
      "identity": {
        "methods": ["application_credential"],
        "application_credential": {
          "id": "'"$OS_APPLICATION_CREDENTIAL_ID"'",
          "secret": "'"$OS_APPLICATION_CREDENTIAL_SECRET"'"
        }
      }
    }
  }' | grep -i "X-Subject-Token" | awk '{print $2}' | tr -d '\r')

# 2. Lister les serveurs via l'API Compute
curl -s -H "X-Auth-Token: $TOKEN" \
  "$NOVA_ENDPOINT/servers" | python3 -m json.tool

Option B — CLI OpenStack

Le client en ligne de commande openstack encapsule les API REST dans des commandes simples et lisibles. C'est l'approche recommandée pour les opérations courantes et celle utilisée tout au long de cette formation. 

# Installer le CLI
pip3 install python-openstackclient

# Exemples
openstack server list
openstack network create my-network
openstack volume create --size 10 my-volume

Option C — OpenStack SDK (Python)

La bibliothèque Python openstacksdk fournit une interface haut niveau et pythonique vers tous les services OpenStack. Elle est idéale pour écrire des scripts et de l'automatisation sur mesure. 

# Installer le SDK
pip3 install openstacksdk
import openstack

# Se connecter via les variables d'environnement (OS_*)
conn = openstack.connect()

# Créer un réseau
network = conn.network.create_network(name="my-network")
print(f"Réseau créé : {network.id}")

# Créer un serveur
server = conn.compute.create_server(
    name="my-instance",
    flavor_id="s1-2",
    image_id="...",
    networks=[{"uuid": network.id}],
)
conn.compute.wait_for_server(server)
print(f"Serveur actif : {server.name}")

Option D — Terraform

Terraform permet de décrire toute votre infrastructure sous forme de code dans des fichiers .tf déclaratifs. Il gère l'ordre des dépendances, le suivi de l'état et les workflows plan/apply. 

# Initialiser et appliquer une configuration Terraform
terraform init
terraform plan
terraform apply

Exemple de fichier main.tf :

# Configurer le provider OpenStack
terraform {
  required_providers {
    openstack = {
      source = "terraform-provider-openstack/openstack"
    }
  }
}

resource "openstack_networking_network_v2" "my_network" {
  name           = "my-network"
  admin_state_up = true
}

resource "openstack_compute_instance_v2" "my_instance" {
  name        = "my-instance"
  flavor_name = "s1-2"
  image_name  = "Ubuntu 24.04"

  network {
    uuid = openstack_networking_network_v2.my_network.id
  }
}

Option E — Ansible

Ansible utilise des playbooks YAML pour automatiser les opérations OpenStack. Il est particulièrement adapté à la gestion de configuration et à l'orchestration de déploiements multi-étapes. 

# Installer la collection Ansible OpenStack
ansible-galaxy collection install openstack.cloud

Exemple de playbook :

- name: Provisionner des ressources OpenStack
  hosts: localhost
  tasks:
    - name: Créer un réseau
      openstack.cloud.network:
        name: my-network
        state: present

    - name: Créer une instance
      openstack.cloud.server:
        name: my-instance
        flavor: s1-2
        image: "Ubuntu 24.04"
        network: my-network
        state: present
        wait: true

3. Quelle approche choisir ?

Approche Idéale pour Courbe d'apprentissage
API REST (cURL) Débogage, compréhension du protocole Élevée
CLI OpenStack Opérations courantes, tâches rapides Faible
OpenStack SDK Scripts Python sur mesure, automatisation Moyenne
Terraform Infrastructure as Code, environnements reproductibles Moyenne
Ansible Gestion de configuration, orchestration Moyenne

Dans cette formation, nous utilisons principalement le CLI OpenStack pour sa simplicité, mais n'hésitez pas à reproduire les exercices avec l'approche de votre choix.

Résumé

  • OpenStack est composé de services indépendants (Nova, Neutron, Cinder, Keystone, Glance).
  • Vous pouvez accéder à l'infrastructure via l'API REST, le CLI OpenStack, le SDK Python, Terraform ou Ansible.
  • Le CLI est le point de départ le plus simple ; Terraform et Ansible sont mieux adaptés à l'infrastructure as code.
  • Toutes les approches s'authentifient via les mêmes variables d'environnement OS_* ou application credentials.
