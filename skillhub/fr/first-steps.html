Premiers pas

Difficulté : Débutant · Durée estimée : 20 min

Avant de plonger dans les services individuels, passons en revue les concepts fondamentaux d'OpenStack et réalisons trois exercices d'introduction : créer une instance, un réseau et un volume.

1. Concepts fondamentaux

Projets

Un projet (aussi appelé tenant) est une unité organisationnelle dans OpenStack. Les ressources comme les instances, réseaux et volumes appartiennent à un projet.

Services

OpenStack est composé de services indépendants :

  • Nova — Calcul (instances de machines virtuelles)
  • Neutron — Réseau (réseaux virtuels, sous-réseaux, routeurs)
  • Cinder — Stockage bloc (volumes, snapshots)
  • Keystone — Identité (authentification, autorisation)
  • Glance — Image (images de machines virtuelles)

Endpoints

Chaque service expose des endpoints HTTP listés dans le catalogue de services Keystone. Les clients découvrent les URLs des services en interrogeant le catalogue après authentification.

2. Exercice 1 — Créer une instance

Utilisez compute_manager pour lancer votre première instance via l'API Nova. Nommez-la {student_id}-first-instance. 

from labs.templates.exercise_template import Exercise

compute = self.managers.get("compute_manager")

instance = compute.create(
    name=f"{student_id}-first-instance",
    flavor="s1-2",
    image="Ubuntu 24.04",
)
print(f"Instance créée : {instance}")

3. Exercice 2 — Créer un réseau

Utilisez network_manager pour créer un réseau virtuel via l'API Neutron. Nommez-le {student_id}-first-network. 

network_mgr = self.managers.get("network_manager")

network = network_mgr.create(
    name=f"{student_id}-first-network",
)
print(f"Réseau créé : {network}")

4. Exercice 3 — Créer un volume

Utilisez volume_manager pour créer un volume de stockage bloc de 1 Go via l'API Cinder. Nommez-le {student_id}-first-volume. 

volume_mgr = self.managers.get("volume_manager")

volume = volume_mgr.create(
    name=f"{student_id}-first-volume",
    size=1,
)
print(f"Volume créé : {volume}")

Résumé

  • OpenStack est composé de services indépendants (Nova, Neutron, Cinder, Keystone, Glance).
  • Chaque exercice utilise un manager fourni par le framework de lab.
  • Les ressources sont nommées avec votre préfixe student_id pour l'isolation.
