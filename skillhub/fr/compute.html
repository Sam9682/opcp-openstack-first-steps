Difficulté : Intermédiaire · Durée estimée : 25 min

Le service de calcul Nova gère les instances de machines virtuelles. Dans cette leçon, vous allez lancer des instances, les redimensionner et créer des snapshots.

1. Paire de clés SSH — Générer et enregistrer

Pour vous connecter à votre instance via SSH, vous avez besoin d'une paire de clés. La clé publique est injectée dans l'instance au démarrage, et vous utilisez la clé privée correspondante pour vous authentifier.

Générer une paire de clés localement

# Générer une paire de clés SSH Ed25519 (recommandé)
ssh-keygen -t ed25519 -C "${STUDENT_ID}@lab" -f ~/.ssh/${STUDENT_ID}-key -N ""

# Cela crée deux fichiers :
#   ~/.ssh/${STUDENT_ID}-key       (clé privée — à garder secrète)
#   ~/.ssh/${STUDENT_ID}-key.pub   (clé publique — envoyée à OpenStack)

Vous pouvez aussi utiliser RSA si Ed25519 n'est pas supporté : 

ssh-keygen -t rsa -b 4096 -C "${STUDENT_ID}@lab" -f ~/.ssh/${STUDENT_ID}-key -N ""

Enregistrer la clé publique dans OpenStack

# Importer votre clé publique dans OpenStack
openstack keypair create \
  --public-key ~/.ssh/${STUDENT_ID}-key.pub \
  ${STUDENT_ID}-keypair

# Vérifier que la keypair est enregistrée
openstack keypair show ${STUDENT_ID}-keypair

2. Exercice 1 — Lancer une instance avec clé SSH

Utilisez le CLI openstack pour lancer une instance avec un flavor, une image et votre paire de clés SSH. Nommez-la ${STUDENT_ID}-compute-instance. 

# Lancer une instance de calcul avec votre clé SSH
openstack server create \
  --flavor scale-1 \
  --image "Ubuntu 24.04 BMPOD" \
  --network Ext-Net \
  --key-name ${STUDENT_ID}-keypair \
  --wait \
  ${STUDENT_ID}-compute-instance

# Vérifier que l'instance est active
openstack server show ${STUDENT_ID}-compute-instance -f value -c status

Se connecter à l'instance via SSH

# Récupérer l'adresse IP de l'instance
IP=$(openstack server show ${STUDENT_ID}-compute-instance -f value -c addresses | grep -oP '[\d.]+')

# Se connecter avec votre clé privée
ssh -i ~/.ssh/${STUDENT_ID}-key ubuntu@$IP

3. Exercice 2 — Redimensionner une instance

Redimensionnez votre instance en cours d'exécution vers un flavor différent. Utile quand votre charge de travail nécessite plus (ou moins) de ressources. 

# Redimensionner l'instance vers un flavor plus grand
openstack server resize --flavor s1-4 ${STUDENT_ID}-compute-instance

# Attendre la fin du redimensionnement, puis confirmer
openstack server resize confirm ${STUDENT_ID}-compute-instance

# Vérifier le nouveau flavor
openstack server show ${STUDENT_ID}-compute-instance -f value -c flavor

4. Exercice 3 — Gérer les snapshots

Créez un snapshot de votre instance. Les snapshots capturent l'état actuel et peuvent être utilisés pour restaurer ou cloner des instances. 

# Créer un snapshot de l'instance
openstack server image create \
  --name ${STUDENT_ID}-compute-snapshot \
  ${STUDENT_ID}-compute-instance

# Vérifier que le snapshot a été créé
openstack image show ${STUDENT_ID}-compute-snapshot -f value -c status

Résumé

  • Générez une paire de clés SSH avec ssh-keygen et enregistrez-la avec openstack keypair create.
  • Utilisez --key-name lors de la création d'une instance pour injecter la clé publique.
  • Utilisez les commandes openstack server pour lancer, redimensionner et snapshotter des instances.
  • Les flavors définissent le CPU, la RAM et le disque alloués à une instance.
  • Les snapshots capturent l'état actuel pour la sauvegarde et la récupération.
  • Les erreurs de flavor ou d'image manquants sont signalées avec des messages clairs par le CLI.
