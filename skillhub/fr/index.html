Introduction aux premiers pas d'OPCP Core OpenStack

Bienvenue sur SkillHub, le compagnon de formation pratique du projet opcp-openstack-first-steps. Ce parcours vous guide à travers les fondamentaux d'OpenStack — de votre premier appel API authentifié à la création d'instances, de réseaux, de volumes et à la configuration de bonds LACP.

Ce que vous apprendrez

À la fin de ce parcours de formation, vous maîtriserez :

  • Les concepts fondamentaux d'OpenStack — projets, utilisateurs, services et endpoints
  • La gestion des utilisateurs — créer des utilisateurs dans Keycloak et attribuer les rôles
  • L'authentification — s'authentifier via Keystone avec des application credentials
  • Le réseau — créer des réseaux, sous-réseaux et routeurs avec Neutron
  • Le stockage — créer et attacher des volumes avec Cinder
  • Le Compute — lancer et gérer des instances avec Nova
  • La configuration LACP — agréger plusieurs interfaces réseau pour la bande passante et la redondance

Prérequis

  • Un projet OVHcloud Public Cloud avec accès API
  • Familiarité de base avec un terminal / ligne de commande
  • Python 3.9+ installé
  • Docker installé (pour l'environnement de lab conteneurisé)

Démarrage rapide — Validez votre configuration

Exécutez ces commandes pour confirmer que votre environnement est prêt :

# Vérifier que le CLI OpenStack est installé
openstack --version

# Vérifier que vos identifiants sont configurés
env | grep OS_

# Tester l'authentification en émettant un token
openstack token issue

Définir votre identifiant étudiant

Tout au long des labs, chaque ressource que vous créez (instances, réseaux, volumes, etc.) est préfixée par un STUDENT_ID. Cela permet d'identifier facilement vos ressources et de les nettoyer après la formation.

Définissez cette variable dans votre terminal avant de commencer un exercice : 

# Définir votre identifiant étudiant (remplacez par le vôtre)
export STUDENT_ID=slepetre

# Vérifier qu'il est défini
echo $STUDENT_ID

Toutes les commandes des labs utilisent ${STUDENT_ID} comme préfixe de nommage. Par exemple, une instance sera nommée ${STUDENT_ID}-compute-instance. À la fin de la formation, le script de nettoyage utilise ce préfixe pour supprimer toutes vos ressources en une seule fois.
