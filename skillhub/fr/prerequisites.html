Prérequis

Difficulté : Débutant · Durée estimée : 15 min

Avant de lancer les labs, assurez-vous que votre environnement remplit les conditions suivantes. Cette page détaille chaque dépendance et comment vérifier qu'elle est correctement installée.

1. Python 3.9+

Le framework de labs et tous les exercices sont écrits en Python. Vous avez besoin de Python 3.9 ou supérieur (3.11 recommandé). 

# Vérifier votre version de Python
python3 --version
# Sortie attendue : Python 3.11.x (ou tout 3.9+)

Si Python n'est pas installé, téléchargez-le depuis python.org ou utilisez votre gestionnaire de paquets : 

# Debian / Ubuntu
sudo apt update && sudo apt install -y python3 python3-pip python3-venv

# macOS (Homebrew)
brew install python@3.11

2. pip (gestionnaire de paquets Python)

pip sert à installer les dépendances Python requises par les labs (openstacksdk, PyYAML, pytest, etc.). 

# Vérifier que pip est disponible
pip3 --version
# Sortie attendue : pip 2x.x from … (python 3.x)

Si pip est absent, installez-le avec :

python3 -m ensurepip --upgrade

3. OpenStack SDK et dépendances Python

Les labs s'appuient sur les paquets Python suivants (définis dans labs/base/requirements.txt) :

  • openstacksdk 3.3.0 — SDK Python officiel pour les API OpenStack
  • PyYAML 6.0.2 — lecture de la configuration YAML
  • pytest 8.3.4 — exécuteur de tests utilisé par le moteur d'évaluation
  • hypothesis 6.112.5 — bibliothèque de tests par propriétés

Installez-les dans un environnement virtuel :

# Créer et activer un environnement virtuel
python3 -m venv .venv
source .venv/bin/activate   # Linux / macOS
# .venv\Scripts\activate    # Windows

# Installer les dépendances
pip install -r labs/base/requirements.txt

4. Docker (optionnel — pour les labs conteneurisés)

Les labs peuvent tourner dans un conteneur Docker qui embarque toutes les dépendances. Si vous préférez cette approche, installez Docker : 

# Vérifier que Docker est installé
docker --version
# Sortie attendue : Docker version 2x.x.x

Installez Docker depuis docs.docker.com.

5. Identifiants OpenStack

Le lanceur de labs a besoin d'identifiants OpenStack valides. Vous pouvez les fournir via des variables d'environnement ou un fichier d'identifiants.

Option A — Variables d'environnement

export OS_AUTH_URL="https://auth.cloud.ovh.net/v3"
export OS_USERNAME="votre-utilisateur"
export OS_PASSWORD="votre-mot-de-passe"
export OS_PROJECT_NAME="votre-projet"
export OS_DOMAIN_NAME="Default"

Option B — Fichier d'identifiants

Créez ~/.openstack/credentials.yaml :

auth_url: https://auth.cloud.ovh.net/v3
username: votre-utilisateur
password: votre-mot-de-passe
project_name: votre-projet
domain_name: Default

6. Projet OVHcloud Public Cloud

Vous avez besoin d'un projet OVHcloud Public Cloud actif avec l'accès API activé. Assurez-vous que votre utilisateur dispose des rôles nécessaires pour créer des instances, réseaux, volumes et groupes de sécurité.

7. Connaissances de base du terminal

Vous devez être à l'aise avec l'exécution de commandes dans un terminal (bash, zsh ou PowerShell). Tous les exercices se lancent en ligne de commande.

Script de validation rapide

Exécutez ce script pour vérifier que tout est en place :

"""Validation des prérequis des labs."""
import shutil
import sys

checks = {
    "Python 3.9+": sys.version_info >= (3, 9),
    "pip disponible": shutil.which("pip3") is not None or shutil.which("pip") is not None,
}

try:
    import openstack  # noqa: F401
    checks["openstacksdk installé"] = True
except ImportError:
    checks["openstacksdk installé"] = False

try:
    import yaml  # noqa: F401
    checks["PyYAML installé"] = True
except ImportError:
    checks["PyYAML installé"] = False

all_ok = True
for name, ok in checks.items():
    status = "OK" if ok else "MANQUANT"
    print(f"  [{status}] {name}")
    if not ok:
        all_ok = False

sys.exit(0 if all_ok else 1)
