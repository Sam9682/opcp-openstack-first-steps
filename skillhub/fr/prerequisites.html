Prérequis

Difficulté : Débutant · Durée estimée : 15 min

Avant de lancer les labs, assurez-vous que votre environnement remplit les conditions suivantes. Cette page détaille chaque dépendance et comment vérifier qu'elle est correctement installée.

1. Python 3.9+

Le framework de labs et tous les exercices sont écrits en Python. Vous avez besoin de Python 3.9 ou supérieur (3.11 recommandé). 

# Vérifier votre version de Python
python3 --version
# Sortie attendue : Python 3.11.x (ou tout 3.9+)

Si Python n'est pas installé, téléchargez-le depuis python.org ou utilisez votre gestionnaire de paquets : 

# Debian / Ubuntu
sudo apt update && sudo apt install -y python3 python3-pip python3-venv

# macOS (Homebrew)
brew install python@3.11

2. pip (gestionnaire de paquets Python)

pip sert à installer les dépendances Python requises par les labs (openstacksdk, PyYAML, pytest, etc.). 

# Vérifier que pip est disponible
pip3 --version
# Sortie attendue : pip 2x.x from … (python 3.x)

Si pip est absent, installez-le avec :

python3 -m ensurepip --upgrade

3. Client CLI OpenStack

Tous les exercices utilisent le client en ligne de commande openstack. Installez-le via pip : 

# Installer le client CLI OpenStack
pip3 install python-openstackclient

# Vérifier l'installation
openstack --version
# Sortie attendue : openstack 7.x.x (ou supérieur)

4. OpenStack SDK et dépendances Python (optionnel)

Si vous souhaitez aussi utiliser le moteur d'évaluation automatisé, installez les paquets Python suivants (définis dans labs/base/requirements.txt) :

  • openstacksdk 3.3.0 — SDK Python officiel pour les API OpenStack
  • PyYAML 6.0.2 — lecture de la configuration YAML
  • pytest 8.3.4 — exécuteur de tests utilisé par le moteur d'évaluation
  • hypothesis 6.112.5 — bibliothèque de tests par propriétés

Installez-les dans un environnement virtuel :

# Créer et activer un environnement virtuel
python3 -m venv .venv
source .venv/bin/activate   # Linux / macOS
# .venv\Scripts\activate    # Windows

# Installer les dépendances
pip install -r labs/base/requirements.txt

4. Docker (optionnel — pour les labs conteneurisés)

Les labs peuvent tourner dans un conteneur Docker qui embarque toutes les dépendances. Si vous préférez cette approche, installez Docker : 

# Vérifier que Docker est installé
docker --version
# Sortie attendue : Docker version 2x.x.x

Installez Docker depuis docs.docker.com.

Installation de Docker sur Ubuntu (si absent)

Si Docker n'est pas installé sur votre machine Ubuntu, suivez les étapes ci-dessous pour l'installer proprement depuis le dépôt officiel Docker : 

# Supprimer d'éventuelles configurations existantes cassées
sudo rm -f /etc/apt/sources.list.d/docker.list
sudo rm -f /etc/apt/sources.list.d/docker.sources

# Ajouter le dépôt Docker avec le codename noble
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  noble stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# S'assurer que la clé GPG existe
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# Mettre à jour et installer Docker
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

5. shai CLI (optionnel — assistant IA en terminal)

shai est un agent de programmation interactif qui vit dans votre terminal. Il peut vous aider à écrire du code, corriger des bugs et répondre à vos questions pendant les labs. 

# Vérifier si shai est installé
shai --version

Si shai n'est pas installé, lancez la commande suivante :

# Installer la dernière version stable
curl -fsSL https://raw.githubusercontent.com/ovh/shai/main/install.sh | sh

Le binaire sera installé dans $HOME/.local/bin. Assurez-vous que ce répertoire est dans votre PATH : 

# Ajouter à votre PATH si nécessaire
export PATH="$HOME/.local/bin:$PATH"

shai-dev — Extension VS Code (alternative)

Si vous préférez utiliser shai directement dans VS Code plutôt qu'en terminal, vous pouvez installer l'extension shai-dev. Elle intègre les fonctionnalités de shai (assistance au code, corrections, questions) directement dans votre éditeur.

Pour l'installer :

  • Ouvrez VS Code, allez dans l'onglet Extensions (Ctrl+Shift+X), recherchez shai-dev et cliquez sur Installer.
  • Ou installez-la en ligne de commande :
code --install-extension shai-dev.shai-vscode

6. Identifiants OpenStack

Le lanceur de labs a besoin d'identifiants OpenStack valides. Vous pouvez les fournir via des variables d'environnement ou un fichier d'identifiants.

Option A — Variables d'environnement

export OS_AUTH_URL="https://auth.cloud.ovh.net/v3"
export OS_USERNAME="votre-utilisateur"
export OS_PASSWORD="votre-mot-de-passe"
export OS_PROJECT_NAME="votre-projet"
export OS_DOMAIN_NAME="Default"

Option B — Fichier d'identifiants

Créez ~/.openstack/credentials.yaml :

auth_url: https://auth.cloud.ovh.net/v3
username: votre-utilisateur
password: votre-mot-de-passe
project_name: votre-projet
domain_name: Default

7. Projet OVHcloud Public Cloud

Vous avez besoin d'un projet OVHcloud Public Cloud actif avec l'accès API activé. Assurez-vous que votre utilisateur dispose des rôles nécessaires pour créer des instances, réseaux, volumes et groupes de sécurité.

8. Connaissances de base du terminal

Vous devez être à l'aise avec l'exécution de commandes dans un terminal (bash, zsh ou PowerShell). Tous les exercices se lancent en ligne de commande.

Script de validation rapide

Exécutez ces commandes pour vérifier que tout est en place :

# Vérifier la version de Python
python3 --version

# Vérifier que pip est disponible
pip3 --version

# Vérifier que le CLI OpenStack est installé
openstack --version

# Tester l'authentification
openstack token issue
