Stockage

Difficulté : Intermédiaire · Durée estimée : 20 min

Le service de stockage bloc Cinder gère les volumes persistants. Dans cette leçon, vous allez créer des volumes, les attacher à des instances et gérer les snapshots de volumes.

1. Exercice 1 — Créer un volume

Utilisez le CLI openstack pour créer un volume de stockage bloc de 1 Go nommé ${STUDENT_ID}-storage-volume. 

# Créer un volume de stockage bloc de 1 Go
openstack volume create \
  --size 1 \
  ${STUDENT_ID}-storage-volume

# Vérifier que le volume est disponible
openstack volume show ${STUDENT_ID}-storage-volume -f value -c status

2. Exercice 2 — Attacher un volume

Attachez votre volume à une instance en cours d'exécution. Le statut du volume passera de available à in-use. 

# Attacher le volume à une instance en cours d'exécution
openstack server add volume \
  ${STUDENT_ID}-compute-instance \
  ${STUDENT_ID}-storage-volume

# Vérifier que le statut du volume est passé à in-use
openstack volume show ${STUDENT_ID}-storage-volume -f value -c status

3. Exercice 3 — Gérer les snapshots

Créez un snapshot de votre volume. Les snapshots de volumes sont des copies à un instant donné qui peuvent servir à créer de nouveaux volumes ou restaurer des données. 

# Créer un snapshot du volume
openstack volume snapshot create \
  --volume ${STUDENT_ID}-storage-volume \
  ${STUDENT_ID}-storage-snapshot

# Vérifier que le snapshot a été créé
openstack volume snapshot show ${STUDENT_ID}-storage-snapshot -f value -c status

Résumé

  • Utilisez les commandes openstack volume pour créer, attacher et snapshotter des volumes.
  • Les volumes sont du stockage bloc persistant qui survit à la suppression d'instance.
  • Attacher un volume change son statut de available à in-use.
  • Les snapshots fournissent des sauvegardes à un instant donné des données du volume.
