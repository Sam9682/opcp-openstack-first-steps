Stockage — Configuration RAID logiciel

Difficulté : Avancé · Durée estimée : 25 min

En environnement de production, s'appuyer sur un seul disque est risqué. Le RAID logiciel (Redundant Array of Independent Disks) combine plusieurs volumes de stockage bloc en un seul périphérique logique pour améliorer les performances, la redondance, ou les deux. Dans cette leçon, vous allez provisionner plusieurs volumes Cinder, les attacher à une instance baremetal et configurer un array RAID logiciel avec mdadm.

1. Contexte — Niveaux de RAID

Les niveaux de RAID logiciel les plus courants sur les instances cloud sont :

NiveauMin. disquesDescriptionCas d'usage
RAID 02 Striping — les données sont réparties sur les disques. Aucune redondance. Débit maximal (espace temporaire, données jetables).
RAID 12 Mirroring — copie identique sur chaque disque. Haute disponibilité pour l'OS ou les données critiques.
RAID 53 Striping avec parité distribuée. Tolère 1 panne de disque. Équilibre entre performances et redondance.
RAID 104 Stripes mirrorés (RAID 1 + 0). Hautes performances avec redondance (bases de données).

2. Exercice 1 — Provisionner plusieurs volumes

Créez les volumes de stockage bloc qui formeront l'array RAID. Nous allons créer 3 volumes de 10 Go chacun pour une configuration RAID 5. 

# Créer 3 volumes pour l'array RAID
for i in 1 2 3; do
  openstack volume create \
    --size 10 \
    ${STUDENT_ID}-raid-disk-${i}
done

# Vérifier que tous les volumes sont disponibles
openstack volume list --format json | jq '.[] | select(.Name | startswith("'$STUDENT_ID'-raid-disk-"))'

3. Exercice 2 — Attacher les volumes à une instance

Attachez les trois volumes à votre instance en cours d'exécution. Chaque volume apparaîtra comme un nouveau périphérique bloc dans l'instance. 

# Attacher chaque volume à l'instance
for i in 1 2 3; do
  openstack server add volume \
    ${STUDENT_ID}-compute-instance \
    ${STUDENT_ID}-raid-disk-${i}
done

# Vérifier que les volumes sont en cours d'utilisation
openstack volume list --name ${STUDENT_ID}-raid-disk-1 -f value -c Status

4. Exercice 3 — Configurer le RAID 5 logiciel

Connectez-vous en SSH à votre instance et utilisez mdadm pour assembler les trois disques en un array RAID 5.

Étape 1 — Identifier les disques attachés

# Lister les périphériques bloc — les nouveaux volumes apparaissent généralement en /dev/vdb, /dev/vdc, /dev/vdd
lsblk

# Sortie attendue (les noms peuvent varier) :
# vdb    10G  disk
# vdc    10G  disk
# vdd    10G  disk

Étape 2 — Installer mdadm

# Installer l'outil de gestion RAID
sudo apt update && sudo apt install -y mdadm

Étape 3 — Créer l'array RAID 5

# Créer un array RAID 5 nommé /dev/md0 avec 3 périphériques
sudo mdadm --create /dev/md0 \
  --level=5 \
  --raid-devices=3 \
  /dev/vdb /dev/vdc /dev/vdd

# Surveiller la progression de la construction
cat /proc/mdstat

Étape 4 — Formater et monter l'array

# Créer un système de fichiers ext4 sur l'array RAID
sudo mkfs.ext4 /dev/md0

# Créer un point de montage et monter l'array
sudo mkdir -p /mnt/raid5
sudo mount /dev/md0 /mnt/raid5

# Vérifier le montage
df -h /mnt/raid5

Étape 5 — Persister la configuration

# Sauvegarder la configuration RAID
sudo mdadm --detail --scan | sudo tee -a /etc/mdadm/mdadm.conf

# Ajouter au fstab pour le montage automatique au redémarrage
echo '/dev/md0 /mnt/raid5 ext4 defaults,nofail 0 2' | sudo tee -a /etc/fstab

# Mettre à jour l'initramfs pour que l'array soit détecté au boot
sudo update-initramfs -u

5. Surveillance et récupération

Une fois l'array en fonctionnement, vous pouvez surveiller son état et simuler une panne de disque pour tester la récupération.

Vérifier l'état de l'array

# État détaillé de l'array RAID
sudo mdadm --detail /dev/md0

# État rapide depuis /proc
cat /proc/mdstat

Simuler une panne de disque

# Marquer un disque comme défaillant
sudo mdadm --manage /dev/md0 --fail /dev/vdd

# Retirer le disque défaillant
sudo mdadm --manage /dev/md0 --remove /dev/vdd

# Vérifier l'état dégradé de l'array
sudo mdadm --detail /dev/md0

Remplacer et reconstruire

# Créer un volume de remplacement dans OpenStack
openstack volume create --size 10 ${STUDENT_ID}-raid-disk-4
openstack server add volume ${STUDENT_ID}-compute-instance ${STUDENT_ID}-raid-disk-4

# Ajouter le nouveau disque à l'array (apparaît en /dev/vde)
sudo mdadm --manage /dev/md0 --add /dev/vde

# Surveiller la reconstruction
watch cat /proc/mdstat

6. Alternative — RAID 1 (Mirroring)

Pour des configurations plus simples nécessitant uniquement du mirroring, vous pouvez créer un array RAID 1 avec deux volumes : 

# Créer 2 volumes
openstack volume create --size 10 ${STUDENT_ID}-mirror-disk-1
openstack volume create --size 10 ${STUDENT_ID}-mirror-disk-2

# Attacher les deux à l'instance
openstack server add volume ${STUDENT_ID}-compute-instance ${STUDENT_ID}-mirror-disk-1
openstack server add volume ${STUDENT_ID}-compute-instance ${STUDENT_ID}-mirror-disk-2

# Créer un array RAID 1 dans l'instance
sudo mdadm --create /dev/md1 \
  --level=1 \
  --raid-devices=2 \
  /dev/vdf /dev/vdg

# Formater et monter
sudo mkfs.ext4 /dev/md1
sudo mkdir -p /mnt/mirror
sudo mount /dev/md1 /mnt/mirror

Résumé

  • Le RAID logiciel utilise mdadm pour combiner plusieurs volumes Cinder en un seul périphérique logique.
  • Le RAID 5 offre un bon équilibre entre performances et redondance grâce à la parité distribuée.
  • Le RAID 1 miroire les données sur deux disques pour une haute disponibilité simple.
  • Utilisez openstack volume create et openstack server add volume pour provisionner et attacher les disques.
  • Persistez toujours la configuration RAID dans /etc/mdadm/mdadm.conf et /etc/fstab.
  • Les disques défaillants peuvent être remplacés et reconstruits sans interruption sur les arrays RAID 1 et RAID 5.
