Networking

Difficulty: Intermediate · Estimated time: 20 min

Networks are the foundation of any cloud deployment. In this lesson you will create virtual networks, configure subnets with CIDR validation, and set up routers — all using the Neutron API through the lab framework.

1. Exercise 1 — Create a Network

Use the openstack CLI to create a virtual network named {student_id}-net-network. 

# Create a virtual network
openstack network create {student_id}-net-network

# Verify the network was created
openstack network show {student_id}-net-network -f value -c status

2. Exercise 2 — Create a Subnet

Create a subnet within your network. Use 10.0.0.0/24 as the address range. 

# Create a subnet with CIDR 10.0.0.0/24
openstack subnet create \
  --network {student_id}-net-network \
  --subnet-range 10.0.0.0/24 \
  {student_id}-net-subnet

# Verify the subnet was created
openstack subnet show {student_id}-net-subnet

3. Exercise 3 — Configure a Router

Create a router and attach it to your subnet. This enables traffic routing between subnets and external networks. 

# Create a router
openstack router create {student_id}-net-router

# Attach the router to your subnet
openstack router add subnet \
  {student_id}-net-router \
  {student_id}-net-subnet

# Verify the router
openstack router show {student_id}-net-router

Summary

  • Use openstack network, openstack subnet, and openstack router commands.
  • Subnets require a valid CIDR range (e.g. 10.0.0.0/24).
  • Routers connect subnets and enable external connectivity.
