Cleanup Resources

Difficulty: Beginner · Estimated time: 10 min

Now that you have completed all the labs, it is time to clean up every OpenStack resource created during the exercises. All lab resources follow the naming convention ${STUDENT_ID}-*, which makes them easy to identify and delete.

1. Naming Convention

Every resource created throughout the labs is prefixed with your student_id. Here is the full list:

ModuleResource typeName pattern
First StepsInstance${STUDENT_ID}-first-instance
First StepsNetwork${STUDENT_ID}-first-network
First StepsVolume${STUDENT_ID}-first-volume
ComputeInstance${STUDENT_ID}-compute-instance
ComputeSnapshot${STUDENT_ID}-compute-snapshot
StorageVolume${STUDENT_ID}-storage-volume
StorageSnapshot${STUDENT_ID}-storage-snapshot
Security GroupsSecurity Group${STUDENT_ID}-sg-web
LACPNetwork${STUDENT_ID}-lacp-network
LACPPorts${STUDENT_ID}-lacp-port-1, ${STUDENT_ID}-lacp-port-2
LACPBond${STUDENT_ID}-lacp-bond
LACPInstance${STUDENT_ID}-lacp-instance
SetupNetworks${STUDENT_ID}-compute-network, ${STUDENT_ID}-storage-network, ${STUDENT_ID}-sg-network
SetupSecurity Group${STUDENT_ID}-default-sg

2. Deletion Order

OpenStack resources have dependencies. Deleting them in the wrong order will fail. The correct order is:

  1. Detach volumes from instances (Cinder).
  2. Delete snapshots — both instance snapshots (Glance/Nova) and volume snapshots (Cinder).
  3. Delete instances (Nova).
  4. Delete bonds and ports (Neutron).
  5. Delete volumes (Cinder).
  6. Delete security groups (Neutron).
  7. Delete networks (Neutron) — subnets are removed automatically.

3. Automated Cleanup

The lab framework provides a cleanup script that deletes all resources matching the ${STUDENT_ID}-* prefix in the correct dependency order: 

# Clean up all resources for your student session
python -m labs.scripts.cleanup_lab --student-id your-student-id

The script uses a best-effort approach: if a single resource fails to delete, it logs the error and continues with the remaining resources.

4. Manual Cleanup with the CLI

You can also clean up resources manually using the openstack CLI. Follow the deletion order above: 

# 1. Detach volumes from instances
openstack server remove volume ${STUDENT_ID}-compute-instance ${STUDENT_ID}-storage-volume

# 2. Delete snapshots (instance and volume)
openstack image delete ${STUDENT_ID}-compute-snapshot
openstack volume snapshot delete ${STUDENT_ID}-storage-snapshot

# 3. Delete instances
openstack server delete ${STUDENT_ID}-first-instance
openstack server delete ${STUDENT_ID}-compute-instance
openstack server delete ${STUDENT_ID}-lacp-instance

# 4. Delete trunks and ports
openstack network trunk delete ${STUDENT_ID}-lacp-bond
openstack port delete ${STUDENT_ID}-lacp-port-1
openstack port delete ${STUDENT_ID}-lacp-port-2

# 5. Delete volumes
openstack volume delete ${STUDENT_ID}-first-volume
openstack volume delete ${STUDENT_ID}-storage-volume

# 6. Delete security groups
openstack security group delete ${STUDENT_ID}-sg-web
openstack security group delete ${STUDENT_ID}-default-sg

# 7. Delete routers (remove interfaces first) and networks
openstack router remove subnet ${STUDENT_ID}-net-router ${STUDENT_ID}-net-subnet
openstack router delete ${STUDENT_ID}-net-router
openstack network delete ${STUDENT_ID}-first-network
openstack network delete ${STUDENT_ID}-net-network
openstack network delete ${STUDENT_ID}-lacp-network
openstack network delete ${STUDENT_ID}-compute-network
openstack network delete ${STUDENT_ID}-storage-network
openstack network delete ${STUDENT_ID}-sg-network

5. Verify Cleanup

After running the cleanup, verify that no resources remain by listing them with the OpenStack CLI: 

# List any remaining instances
openstack server list --name "your-student-id-*"

# List any remaining volumes
openstack volume list --name "your-student-id-*"

# List any remaining networks
openstack network list --name "your-student-id-*"

# List any remaining security groups
openstack security group list --project your-project-id | grep "your-student-id"

If any resources remain, delete them manually using the openstack CLI or re-run the cleanup script.
