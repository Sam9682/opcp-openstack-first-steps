User Management

Difficulty: Beginner · Estimated time: 20 min

Before authenticating against OpenStack, you need a user account. OPCP Core relies on Keycloak as its identity provider. This lesson walks you through creating users and assigning them the right roles and project attributes.

1. Built-in Roles

After the very first installation of OPCP, a super-admin account (login / password) is communicated to you. This account is used to connect to the Keycloak UI (or API) and manage users.

Keycloak realms, roles, and groups are pre-configured out of the box. The OPCP solution is delivered with the following built-in roles:

  • Master Admin role — the pre-configured super-admin delivered with OPCP.
  • IT Admin — administers OpenStack projects and manages resources for end-users.
  • DC Operator — operates the data-center infrastructure layer.

2. Creating a User in Keycloak

Log in to the Keycloak administration console with the super-admin credentials, then navigate to Users → Add user. Fill in the required fields (username, email, first / last name) and save.

Setting a Password

After creating the user, open the Credentials tab and click Set Password. Enter the initial password. Enable the Temporary toggle if you want the user to be prompted to reset their password at first login.

3. Configuring an IT Admin User

When you need to create an IT Admin — someone who administers OpenStack projects and manages resources for end-users — you must add a project attribute to the user.

Open the Attributes tab of the user and add a key-value pair:

  • Key: project
  • Value: a single-line JSON object (see below)
{"domain": {"name": "Default"}, "name": "admin", "roles": [{"name": "admin"}, {"name": "service"}]}

Adapt the values to match your setup:

  • domain → name — the OpenStack domain (e.g. Default).
  • name — the OpenStack project name (e.g. admin).
  • roles — the OpenStack roles to grant (e.g. admin, service).

4. Configuring an End-User

End-users need access to specific projects on OpenStack within a restricted domain. After creating the user, open the Attributes tab and add a key-value pair:

  • Key: project
  • Value: a single-line JSON object (see below)
{"domain": {"name": "keycloak"}, "name": "$project", "roles": [{"name": "member"}, {"name": "reader"}]}

The keycloak domain is the restricted domain accessible to end-users. Replace $project with the actual OpenStack project name the user should have access to.

Summary

  • OPCP ships with pre-configured Keycloak realms, roles, and groups.
  • Three built-in roles: Master Admin role, IT Admin, and DC Operator.
  • User passwords are set in the Credentials tab; enable Temporary to force a reset on first login.
  • IT Admins receive a project attribute on the Default domain with admin / service roles.
  • End-users receive a project attribute on the keycloak domain with member / reader roles.
