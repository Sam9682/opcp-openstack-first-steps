First Steps

Difficulty: Beginner · Estimated time: 20 min

Before diving into individual services, let's review the foundational concepts of OpenStack and run through three introductory exercises: creating an instance, a network, and a volume.

1. Core Concepts

Projects

A project (also called a tenant) is an organizational unit in OpenStack. Resources such as instances, networks, and volumes belong to a project.

Services

OpenStack is composed of independent services:

  • Nova — Compute (virtual machine instances)
  • Neutron — Networking (virtual networks, subnets, routers)
  • Cinder — Block Storage (volumes, snapshots)
  • Keystone — Identity (authentication, authorization)
  • Glance — Image (virtual machine images)

Endpoints

Each service exposes HTTP endpoints listed in the Keystone service catalog. Clients discover service URLs by querying the catalog after authentication.

2. Exercise 1 — Create an Instance

Use compute_manager to launch your first compute instance via the Nova API. Name it {student_id}-first-instance. 

from labs.templates.exercise_template import Exercise

# The exercise framework provides managers via self.managers
compute = self.managers.get("compute_manager")

instance = compute.create(
    name=f"{student_id}-first-instance",
    flavor="s1-2",        # from config
    image="Ubuntu 24.04", # from config
)
print(f"Instance created: {instance}")

3. Exercise 2 — Create a Network

Use network_manager to create a virtual network via the Neutron API. Name it {student_id}-first-network. 

network_mgr = self.managers.get("network_manager")

network = network_mgr.create(
    name=f"{student_id}-first-network",
)
print(f"Network created: {network}")

4. Exercise 3 — Create a Volume

Use volume_manager to create a 1 GB block storage volume via the Cinder API. Name it {student_id}-first-volume. 

volume_mgr = self.managers.get("volume_manager")

volume = volume_mgr.create(
    name=f"{student_id}-first-volume",
    size=1,  # 1 GB
)
print(f"Volume created: {volume}")

Summary

  • OpenStack is composed of independent services (Nova, Neutron, Cinder, Keystone, Glance).
  • Each exercise uses a manager provided by the lab framework.
  • Resources are named with your student_id prefix for isolation.
