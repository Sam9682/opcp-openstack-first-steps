First Steps

Difficulty: Beginner · Estimated time: 20 min

Before diving into individual services, let's review the foundational concepts of OpenStack and run through three introductory exercises: creating an instance, a network, and a volume.

1. Core Concepts

Projects

A project (also called a tenant) is an organizational unit in OpenStack. Resources such as instances, networks, and volumes belong to a project.

Services

OpenStack is composed of independent services:

  • Nova — Compute (virtual machine instances)
  • Neutron — Networking (virtual networks, subnets, routers)
  • Cinder — Block Storage (volumes, snapshots)
  • Keystone — Identity (authentication, authorization)
  • Glance — Image (virtual machine images)

Endpoints

Each service exposes HTTP endpoints listed in the Keystone service catalog. Clients discover service URLs by querying the catalog after authentication.

2. Exercise 1 — Create an Instance

Use the openstack CLI to launch your first compute instance via the Nova API. Name it {student_id}-first-instance. 

# Create a compute instance with the openstack CLI
openstack server create \
  --flavor s1-2 \
  --image "Ubuntu 24.04" \
  --network Ext-Net \
  --wait \
  {student_id}-first-instance

# Verify the instance is active
openstack server show {student_id}-first-instance -f value -c status

3. Exercise 2 — Create a Network

Use the openstack CLI to create a virtual network via the Neutron API. Name it {student_id}-first-network. 

# Create a virtual network
openstack network create {student_id}-first-network

# Verify the network was created
openstack network show {student_id}-first-network -f value -c status

4. Exercise 3 — Create a Volume

Use the openstack CLI to create a 1 GB block storage volume via the Cinder API. Name it {student_id}-first-volume. 

# Create a 1 GB block storage volume
openstack volume create \
  --size 1 \
  {student_id}-first-volume

# Verify the volume is available
openstack volume show {student_id}-first-volume -f value -c status

Summary

  • OpenStack is composed of independent services (Nova, Neutron, Cinder, Keystone, Glance).
  • Each exercise uses the openstack CLI to interact directly with the APIs.
  • Resources are named with your student_id prefix for isolation.
