CLI & API Cheat Sheet

Difficulty: Beginner · Estimated time: 10 min

This page lists common OpenStack operations using both the openstack CLI and the equivalent curl calls against the REST API. Replace the placeholder values ($OS_AUTH_URL, $TOKEN, etc.) with your own environment variables.

Authentication — Get a Token

CLI

openstack token issue

curl — Application Credentials

curl -s -X POST "$OS_AUTH_URL/auth/tokens" \
  -H "Content-Type: application/json" \
  -d '{
    "auth": {
      "identity": {
        "methods": ["application_credential"],
        "application_credential": {
          "id": "'"$OS_APPLICATION_CREDENTIAL_ID"'",
          "secret": "'"$OS_APPLICATION_CREDENTIAL_SECRET"'"
        }
      }
    }
  }' -i | grep "X-Subject-Token"

curl — User / Password

curl -s -X POST "$OS_AUTH_URL/auth/tokens" \
  -H "Content-Type: application/json" \
  -d '{
    "auth": {
      "identity": {
        "methods": ["password"],
        "password": {
          "user": {
            "name": "'"$OS_USERNAME"'",
            "domain": {"name": "'"$OS_DOMAIN_NAME"'"},
            "password": "'"$OS_PASSWORD"'"
          }
        }
      },
      "scope": {
        "project": {
          "name": "'"$OS_PROJECT_NAME"'",
          "domain": {"name": "'"$OS_DOMAIN_NAME"'"}
        }
      }
    }
  }' -i | grep "X-Subject-Token"

Compute (Nova)

List servers — CLI

openstack server list

List servers — curl

curl -s -X GET "$NOVA_ENDPOINT/servers" \
  -H "X-Auth-Token: $TOKEN" | python3 -m json.tool

Create a server — CLI

openstack server create \
  --flavor s1-4 \
  --image "Ubuntu 22.04" \
  --network my-network \
  --wait \
  my-instance

Create a server — curl

curl -s -X POST "$NOVA_ENDPOINT/servers" \
  -H "X-Auth-Token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "server": {
      "name": "my-instance",
      "flavorRef": "'"$FLAVOR_ID"'",
      "imageRef": "'"$IMAGE_ID"'",
      "networks": [{"uuid": "'"$NETWORK_ID"'"}]
    }
  }' | python3 -m json.tool

Delete a server — CLI

openstack server delete my-instance

Delete a server — curl

curl -s -X DELETE "$NOVA_ENDPOINT/servers/$SERVER_ID" \
  -H "X-Auth-Token: $TOKEN"

Networking (Neutron)

List networks — CLI

openstack network list

List networks — curl

curl -s -X GET "$NEUTRON_ENDPOINT/v2.0/networks" \
  -H "X-Auth-Token: $TOKEN" | python3 -m json.tool

Create a network — CLI

openstack network create my-network

Create a network — curl

curl -s -X POST "$NEUTRON_ENDPOINT/v2.0/networks" \
  -H "X-Auth-Token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "network": {
      "name": "my-network",
      "admin_state_up": true
    }
  }' | python3 -m json.tool

Create a subnet — CLI

openstack subnet create \
  --network my-network \
  --subnet-range 192.168.1.0/24 \
  my-subnet

Create a subnet — curl

curl -s -X POST "$NEUTRON_ENDPOINT/v2.0/subnets" \
  -H "X-Auth-Token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "subnet": {
      "network_id": "'"$NETWORK_ID"'",
      "ip_version": 4,
      "cidr": "192.168.1.0/24",
      "name": "my-subnet"
    }
  }' | python3 -m json.tool

Storage (Cinder)

List volumes — CLI

openstack volume list

List volumes — curl

curl -s -X GET "$CINDER_ENDPOINT/volumes" \
  -H "X-Auth-Token: $TOKEN" | python3 -m json.tool

Create a volume — CLI

openstack volume create --size 10 my-volume

Create a volume — curl

curl -s -X POST "$CINDER_ENDPOINT/volumes" \
  -H "X-Auth-Token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "volume": {
      "size": 10,
      "name": "my-volume"
    }
  }' | python3 -m json.tool

Attach a volume to a server — CLI

openstack server add volume my-instance my-volume

Attach a volume to a server — curl

curl -s -X POST "$NOVA_ENDPOINT/servers/$SERVER_ID/os-volume_attachments" \
  -H "X-Auth-Token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "volumeAttachment": {
      "volumeId": "'"$VOLUME_ID"'"
    }
  }' | python3 -m json.tool

Useful Tips

  • Store your token in a variable for reuse: 
    export TOKEN=$(openstack token issue -f value -c id)
  • Find service endpoints: 
    openstack catalog list
  • Use -f json with the CLI to get JSON output, handy for scripting: 
    openstack server list -f json
