LACP Configuration

Difficulty: Advanced · Estimated time: 30 min

Link Aggregation Control Protocol (LACP) allows you to bond multiple physical network interfaces into a single logical link, providing increased bandwidth and redundancy. In this lesson you will configure LACP bonds on OpenStack using the Neutron API through the lab framework.

Background — What is LACP?

LACP (IEEE 802.3ad) negotiates link aggregation between two endpoints automatically. In an OpenStack context this is typically used to bond ports on bare-metal or SR-IOV instances, or to configure trunk ports that carry multiple VLANs over an aggregated link.

  • Active mode — the interface sends LACPDU frames to initiate negotiation.
  • Passive mode — the interface responds to LACPDU frames but does not initiate.
  • Bond modes — balance-rr, active-backup, balance-xor, 802.3ad (LACP).

1. Exercise 1 — Create a LACP Bond Port

Use the openstack CLI to create two Neutron ports on the same network, then create a trunk port that aggregates them using LACP (802.3ad bond mode). 

# Create the base network
openstack network create {student_id}-lacp-network

# Create two member ports on the network
openstack port create \
  --network {student_id}-lacp-network \
  {student_id}-lacp-port-1

openstack port create \
  --network {student_id}-lacp-network \
  {student_id}-lacp-port-2

# Create a trunk port using the first port as parent
openstack network trunk create \
  --parent-port {student_id}-lacp-port-1 \
  --subport port={student_id}-lacp-port-2,segmentation-type=vlan,segmentation-id=100 \
  {student_id}-lacp-bond

# Verify the trunk was created
openstack network trunk show {student_id}-lacp-bond

2. Exercise 2 — Configure LACP Parameters

Fine-tune the LACP bond by updating the trunk port configuration. These parameters control how the bond negotiates and distributes traffic. 

# Add another subport to the trunk
openstack network trunk set \
  --subport port={student_id}-lacp-port-2,segmentation-type=vlan,segmentation-id=200 \
  {student_id}-lacp-bond

# Verify the trunk configuration
openstack network trunk show {student_id}-lacp-bond

3. Exercise 3 — Attach LACP Bond to an Instance

Attach the LACP bond to a compute instance so the instance benefits from the aggregated bandwidth and link redundancy. 

# Launch an instance using the trunk parent port
openstack server create \
  --flavor s1-4 \
  --image "ubuntu-22.04" \
  --port {student_id}-lacp-port-1 \
  --wait \
  {student_id}-lacp-instance

# Verify the instance is active and connected
openstack server show {student_id}-lacp-instance -f value -c status
openstack port show {student_id}-lacp-port-1 -f value -c status

Summary

  • LACP (802.3ad) bonds multiple ports into a single logical link.
  • Use openstack network trunk commands to create and manage trunk ports.
  • Subports are added with --subport specifying segmentation type and ID.
  • Instances connect to trunks via the parent port using --port.
