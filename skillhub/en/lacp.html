LACP Configuration

Difficulty: Advanced · Estimated time: 30 min

Link Aggregation Control Protocol (LACP) allows you to bond multiple physical network interfaces into a single logical link, providing increased bandwidth and redundancy. In this lesson you will configure LACP bonds on OpenStack using the Neutron API through the lab framework.

Background — What is LACP?

LACP (IEEE 802.3ad) negotiates link aggregation between two endpoints automatically. In an OpenStack context this is typically used to bond ports on bare-metal or SR-IOV instances, or to configure trunk ports that carry multiple VLANs over an aggregated link.

  • Active mode — the interface sends LACPDU frames to initiate negotiation.
  • Passive mode — the interface responds to LACPDU frames but does not initiate.
  • Bond modes — balance-rr, active-backup, balance-xor, 802.3ad (LACP).

1. Exercise 1 — Create a LACP Bond Port

Use network_manager to create two Neutron ports on the same network, then create a trunk port that aggregates them using LACP (802.3ad bond mode). 

net = self.managers.get("network_manager")

# Create the base network
network = net.create(name=f"{student_id}-lacp-network")

# Create two member ports
port1 = net.create_port(
    network_name=f"{student_id}-lacp-network",
    name=f"{student_id}-lacp-port-1",
)
port2 = net.create_port(
    network_name=f"{student_id}-lacp-network",
    name=f"{student_id}-lacp-port-2",
)

# Create the LACP bond
bond = net.create_bond(
    name=f"{student_id}-lacp-bond",
    port_names=[f"{student_id}-lacp-port-1", f"{student_id}-lacp-port-2"],
    bond_mode="802.3ad",
)
print(f"LACP bond created: {bond}")

2. Exercise 2 — Configure LACP Parameters

Fine-tune the LACP bond by setting the LACP rate, transmit hash policy, and LACP activity mode. These parameters control how the bond negotiates and distributes traffic. 

# Update LACP parameters on the bond
net.update_bond(
    bond_name=f"{student_id}-lacp-bond",
    lacp_rate="fast",              # fast = 1s, slow = 30s
    transmit_hash_policy="layer3+4",  # hash on IP + port
    lacp_activity="active",        # active LACPDU negotiation
)
print("LACP parameters configured")

3. Exercise 3 — Attach LACP Bond to an Instance

Attach the LACP bond to a compute instance so the instance benefits from the aggregated bandwidth and link redundancy. 

compute = self.managers.get("compute_manager")

# Launch an instance
instance = compute.create(
    name=f"{student_id}-lacp-instance",
    image="ubuntu-22.04",
    flavor="s1-4",
)

# Attach the bond to the instance
net.attach_bond_to_instance(
    bond_name=f"{student_id}-lacp-bond",
    instance_name=f"{student_id}-lacp-instance",
)
print(f"Bond attached to instance: {instance}")

Summary

  • LACP (802.3ad) bonds multiple ports into a single logical link.
  • Use network_manager to create ports, bonds, and configure LACP parameters.
  • Key parameters: lacp_rate (fast/slow), transmit_hash_policy, and lacp_activity (active/passive).
  • Bonds can be attached to instances for increased bandwidth and failover.
