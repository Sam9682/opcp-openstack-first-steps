Storage — Software RAID Configuration

Difficulty: Advanced · Estimated time: 25 min

In production environments, relying on a single disk is risky. Software RAID (Redundant Array of Independent Disks) combines multiple block storage volumes into a single logical device to improve performance, redundancy, or both. In this lesson you will provision multiple Cinder volumes, attach them to a baremetal instance, and configure a software RAID array using mdadm.

1. Background — RAID Levels

The most common software RAID levels used on cloud instances are:

LevelMin. disksDescriptionUse case
RAID 02 Striping — data is split across disks. No redundancy. Maximum throughput (scratch space, temp data).
RAID 12 Mirroring — identical copy on each disk. High availability for OS or critical data.
RAID 53 Striping with distributed parity. Tolerates 1 disk failure. Balanced performance and redundancy.
RAID 104 Mirrored stripes (RAID 1 + 0). High performance with redundancy (databases).

2. Exercise 1 — Provision Multiple Volumes

Create the block storage volumes that will form the RAID array. We will create 3 volumes of 10 GB each for a RAID 5 configuration. 

# Create 3 volumes for the RAID array
for i in 1 2 3; do
  openstack volume create \
    --size 10 \
    ${STUDENT_ID}-raid-disk-${i}
done

# Verify all volumes are available
openstack volume list --format json | jq '.[] | select(.Name | startswith("'$STUDENT_ID'-raid-disk-"))'

3. Exercise 2 — Attach Volumes to an Instance

Attach all three volumes to your running instance. Each volume will appear as a new block device inside the instance. 

# Attach each volume to the instance
for i in 1 2 3; do
  openstack server add volume \
    ${STUDENT_ID}-compute-instance \
    ${STUDENT_ID}-raid-disk-${i}
done

# Verify volumes are in-use
openstack volume list --name ${STUDENT_ID}-raid-disk-1 -f value -c Status

4. Exercise 3 — Configure Software RAID 5

SSH into your instance and use mdadm to assemble the three disks into a RAID 5 array.

Step 1 — Identify the attached disks

# List block devices — the new volumes typically appear as /dev/vdb, /dev/vdc, /dev/vdd
lsblk

# Expected output (names may vary):
# vdb    10G  disk
# vdc    10G  disk
# vdd    10G  disk

Step 2 — Install mdadm

# Install the RAID management tool
sudo apt update && sudo apt install -y mdadm

Step 3 — Create the RAID 5 array

# Create a RAID 5 array named /dev/md0 with 3 devices
sudo mdadm --create /dev/md0 \
  --level=5 \
  --raid-devices=3 \
  /dev/vdb /dev/vdc /dev/vdd

# Monitor the array build progress
cat /proc/mdstat

Step 4 — Format and mount the array

# Create an ext4 filesystem on the RAID array
sudo mkfs.ext4 /dev/md0

# Create a mount point and mount the array
sudo mkdir -p /mnt/raid5
sudo mount /dev/md0 /mnt/raid5

# Verify the mount
df -h /mnt/raid5

Step 5 — Persist the configuration

# Save the RAID configuration
sudo mdadm --detail --scan | sudo tee -a /etc/mdadm/mdadm.conf

# Add to fstab for automatic mount on reboot
echo '/dev/md0 /mnt/raid5 ext4 defaults,nofail 0 2' | sudo tee -a /etc/fstab

# Update initramfs so the array is detected at boot
sudo update-initramfs -u

5. Monitoring and Recovery

Once the array is running, you can monitor its health and simulate a disk failure to test recovery.

Check array status

# Detailed status of the RAID array
sudo mdadm --detail /dev/md0

# Quick status from /proc
cat /proc/mdstat

Simulate a disk failure

# Mark a disk as faulty
sudo mdadm --manage /dev/md0 --fail /dev/vdd

# Remove the faulty disk
sudo mdadm --manage /dev/md0 --remove /dev/vdd

# Check the degraded array status
sudo mdadm --detail /dev/md0

Replace and rebuild

# Create a replacement volume in OpenStack
openstack volume create --size 10 ${STUDENT_ID}-raid-disk-4
openstack server add volume ${STUDENT_ID}-compute-instance ${STUDENT_ID}-raid-disk-4

# Add the new disk to the array (appears as /dev/vde)
sudo mdadm --manage /dev/md0 --add /dev/vde

# Monitor the rebuild
watch cat /proc/mdstat

6. Alternative — RAID 1 (Mirroring)

For simpler setups requiring only mirroring, you can create a RAID 1 array with two volumes: 

# Create 2 volumes
openstack volume create --size 10 ${STUDENT_ID}-mirror-disk-1
openstack volume create --size 10 ${STUDENT_ID}-mirror-disk-2

# Attach both to the instance
openstack server add volume ${STUDENT_ID}-compute-instance ${STUDENT_ID}-mirror-disk-1
openstack server add volume ${STUDENT_ID}-compute-instance ${STUDENT_ID}-mirror-disk-2

# Create a RAID 1 array inside the instance
sudo mdadm --create /dev/md1 \
  --level=1 \
  --raid-devices=2 \
  /dev/vdf /dev/vdg

# Format and mount
sudo mkfs.ext4 /dev/md1
sudo mkdir -p /mnt/mirror
sudo mount /dev/md1 /mnt/mirror

Summary

  • Software RAID uses mdadm to combine multiple Cinder volumes into a single logical device.
  • RAID 5 provides a good balance of performance and redundancy with distributed parity.
  • RAID 1 mirrors data across two disks for simple high availability.
  • Use openstack volume create and openstack server add volume to provision and attach disks.
  • Always persist the RAID configuration in /etc/mdadm/mdadm.conf and /etc/fstab.
  • Failed disks can be replaced and rebuilt without downtime on RAID 1 and RAID 5 arrays.
