Storage

Difficulty: Intermediate · Estimated time: 20 min

The Cinder block storage service manages persistent volumes. In this lesson you will create volumes, attach them to running instances, and manage volume snapshots.

1. Exercise 1 — Create a Volume

Use the openstack CLI to create a 1 GB block storage volume named {student_id}-storage-volume. 

# Create a 1 GB block storage volume
openstack volume create \
  --size 1 \
  {student_id}-storage-volume

# Verify the volume is available
openstack volume show {student_id}-storage-volume -f value -c status

2. Exercise 2 — Attach a Volume

Attach your volume to a running compute instance. The volume status will change from available to in-use. 

# Attach the volume to a running instance
openstack server add volume \
  {student_id}-compute-instance \
  {student_id}-storage-volume

# Verify the volume status changed to in-use
openstack volume show {student_id}-storage-volume -f value -c status

3. Exercise 3 — Manage Snapshots

Create a snapshot of your volume. Volume snapshots are point-in-time copies that can be used to create new volumes or restore data. 

# Create a snapshot of the volume
openstack volume snapshot create \
  --volume {student_id}-storage-volume \
  {student_id}-storage-snapshot

# Verify the snapshot was created
openstack volume snapshot show {student_id}-storage-snapshot -f value -c status

Summary

  • Use openstack volume commands to create, attach, and snapshot volumes.
  • Volumes are persistent block storage that survive instance deletion.
  • Attaching a volume changes its status from available to in-use.
  • Snapshots provide point-in-time backups of volume data.
