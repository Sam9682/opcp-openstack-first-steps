Security Groups

Difficulty: Intermediate · Estimated time: 20 min

Security groups act as virtual firewalls for your instances. In this lesson you will create security groups, manage ingress/egress rules, and apply them to running instances.

1. Exercise 1 — Create a Security Group

Use security_group_manager to create a security group named {student_id}-sg-web. 

sg = self.managers.get("security_group_manager")

group = sg.create(
    name=f"{student_id}-sg-web",
    description="Web traffic security group",
)
print(f"Security group created: {group}")

2. Exercise 2 — Manage Rules

Add ingress rules to allow HTTP (port 80) and HTTPS (port 443) traffic. Rules are validated before being sent to the API. 

from labs.core.validators import validate_sg_rule

# Validate the rule parameters
valid, err = validate_sg_rule("tcp", 443, 443)
if not valid:
    print(f"Validation failed: {err}")

# Add HTTPS ingress rule
sg.add_rule(
    f"{student_id}-sg-web",
    protocol="tcp",
    port_min=443,
    port_max=443,
    direction="ingress",
)
print("HTTPS rule added")

3. Exercise 3 — Apply to an Instance

Apply your security group to a running compute instance. This controls which network traffic can reach the instance. 

sg.apply_to_instance(
    sg_name=f"{student_id}-sg-web",
    instance_name=f"{student_id}-compute-instance",
)
print("Security group applied to instance")

Summary

  • Security groups are virtual firewalls that control network access.
  • Rules specify protocol, port range, and direction (ingress/egress).
  • Rules are validated before being sent to the API.
  • Security groups are applied to instances to enforce network policies.
