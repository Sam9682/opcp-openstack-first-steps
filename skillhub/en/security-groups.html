Security Groups

Difficulty: Intermediate · Estimated time: 20 min

Security groups act as virtual firewalls for your instances. In this lesson you will create security groups, manage ingress/egress rules, and apply them to running instances.

1. Exercise 1 — Create a Security Group

Use the openstack CLI to create a security group named {student_id}-sg-web. 

# Create a security group
openstack security group create \
  --description "Web traffic security group" \
  {student_id}-sg-web

# Verify the security group was created
openstack security group show {student_id}-sg-web

2. Exercise 2 — Manage Rules

Add ingress rules to allow HTTP (port 80) and HTTPS (port 443) traffic. 

# Add an HTTP ingress rule (port 80)
openstack security group rule create \
  --protocol tcp \
  --dst-port 80 \
  --ingress \
  {student_id}-sg-web

# Add an HTTPS ingress rule (port 443)
openstack security group rule create \
  --protocol tcp \
  --dst-port 443 \
  --ingress \
  {student_id}-sg-web

# List the rules to verify
openstack security group rule list {student_id}-sg-web

3. Exercise 3 — Apply to an Instance

Apply your security group to a running compute instance. This controls which network traffic can reach the instance. 

# Add the security group to a running instance
openstack server add security group \
  {student_id}-compute-instance \
  {student_id}-sg-web

# Verify the security group is attached
openstack server show {student_id}-compute-instance -f value -c security_groups

Summary

  • Security groups are virtual firewalls that control network access.
  • Rules specify protocol, port range, and direction (ingress/egress).
  • Use openstack security group rule create to add rules.
  • Security groups are applied to instances with openstack server add security group.
