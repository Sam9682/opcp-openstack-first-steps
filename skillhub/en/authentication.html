Authentication

Difficulty: Beginner · Estimated time: 15 min

Before managing any OpenStack resource you need a valid, authenticated connection. The Keystone identity service handles authentication and issues tokens that authorize subsequent API calls. In this course we use application credentials — a safer alternative to user credentials that can be scoped and revoked independently.

1. What Are Application Credentials?

Application credentials are project-scoped secrets created from your OpenStack user account. They let scripts and automation authenticate without exposing your main username and password. Key benefits:

  • Can be restricted to specific roles and access rules
  • Revocable at any time without changing your user password
  • Ideal for CI/CD pipelines, lab environments, and automation

2. Environment Variables

Export your application credential values so the lab framework can authenticate automatically.

export OS_AUTH_URL="https://auth.cloud.ovh.net/v3"
export OS_AUTH_TYPE="v3applicationcredential"
export OS_APPLICATION_CREDENTIAL_ID="${OS_APPLICATION_CREDENTIAL_ID}"
export OS_APPLICATION_CREDENTIAL_SECRET="${OS_APPLICATION_CREDENTIAL_SECRET}"
export OS_REGION_NAME="GRA7"

3. Exercise 1 — Authenticate with Application Credentials

Use the openstack CLI to authenticate against Keystone using your application credential and obtain a token. 

# Request a token using application credentials
openstack token issue

# The command outputs a token ID and expiration date
# The CLI uses the OS_* environment variables automatically

4. Exercise 2 — Token Management

Learn to validate and revoke authentication tokens. This is essential for understanding token lifecycle in long-running automation. 

# Issue a token and capture its ID
TOKEN=$(openstack token issue -f value -c id)
echo "Token obtained: $TOKEN"

# Revoke a token you no longer need
openstack token revoke "$TOKEN"
echo "Token revoked successfully"

5. Error Handling

Authentication failures are reported with clear error messages by the CLI. If your credentials are wrong, the CLI will return an error. 

# Example: attempting authentication with wrong credentials
# will produce an error like:
#   The request you have made requires authentication. (HTTP 401)

# Always verify your environment variables are set correctly
env | grep OS_

Summary

  • Use application credentials instead of user credentials for safer automation.
  • Set OS_AUTH_TYPE=v3applicationcredential along with your credential ID and secret.
  • Use openstack token issue to obtain a token and openstack token revoke to revoke it.
  • The CLI reads OS_* environment variables automatically for authentication.
  • Authentication errors are reported clearly by the CLI.
